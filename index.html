Introduction to Regular Expressions

Regular expressions (regex) are powerful tools used to search, match, and manipulate strings of text. They consist of a combination of literal characters and special metacharacters that define patterns.

Example: The regex /hello/ matches the word "hello" in any string of text.

In this guide, you will learn how to use regex for various tasks like validating input, searching text, and more.

Basic Syntax

Regex patterns are composed of basic elements that include literals, special characters, and more. Here's an overview of the basic syntax:

  • a - Matches the character 'a' literally.
  • abc - Matches the exact sequence "abc".
  • . - Matches any character except a newline.
  • \d - Matches any digit (0-9).
  • \w - Matches any word character (a-z, A-Z, 0-9, _).
  • \s - Matches any whitespace character (space, tab, newline).

Example: The regex \\d{3}-\\d{3}-\\d{4} matches phone numbers in the format 123-456-7890.

Meta Characters

Meta characters have special meanings in regex and are essential for constructing patterns. Some of the most common meta characters include:

  • . - Matches any single character except newline.
  • ^ - Anchors a match at the beginning of the string.
  • $ - Anchors a match at the end of the string.
  • \b - Matches a word boundary.
  • [ ] - Matches any character within the brackets.
  • [^ ] - Matches any character not within the brackets.

Example: The regex \\bword\\b matches the word "word" as a whole word.

Character Classes

Character classes allow you to define sets of characters to match. Some examples include:

  • [a-z] - Matches any lowercase letter from a to z.
  • [A-Z] - Matches any uppercase letter from A to Z.
  • [0-9] - Matches any digit from 0 to 9.
  • [^a-z] - Matches any character that is not a lowercase letter.
  • [a-zA-Z0-9] - Matches any alphanumeric character.

Quantifiers

Quantifiers define how many times a pattern should match. Some common quantifiers include:

  • * - Matches 0 or more occurrences of the preceding element.
  • + - Matches 1 or more occurrences of the preceding element.
  • ? - Matches 0 or 1 occurrence of the preceding element.
  • {n} - Matches exactly n occurrences of the preceding element.
  • {n,} - Matches at least n occurrences of the preceding element.
  • {n,m} - Matches between n and m occurrences of the preceding element.

Anchors

Anchors are used to match the position within a string:

  • ^ - Matches the start of a string.
  • $ - Matches the end of a string.

Example: The regex ^abc matches any string that starts with "abc".

Examples

Here are some practical regex examples:

  • Match an email address: /\\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Z]{2,}\\b/
  • Match a phone number: /\\d{3}-\\d{3}-\\d{4}/
  • Match a date in the format YYYY-MM-DD: /\\d{4}-\\d{2}-\\d{2}/
  • Match a URL: /https?:\\/\\/(www\\.)?[a-z0-9]+\\.[a-z]{2,3}/

Regex Reference

Here is a quick reference for some commonly used regex patterns:

  • \\d - Matches any digit
  • \\w - Matches any alphanumeric character (letters, digits, and underscore)
  • \\s - Matches any whitespace character (space, tab, newline)
  • ^ - Start of string
  • $ - End of string
  • .* - Matches any character (except newlines) 0 or more times
